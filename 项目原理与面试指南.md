# 项目原理与面试指南：从零基础到面试通关

这份文档专为“小白”视角设计，旨在通过本项目（留学申请系统）为你讲解 Web 开发的核心原理、运行逻辑以及常见的面试考点。

---

## 第一章：宏观架构 —— 我们的系统长什么样？

### 1.1 核心概念：B/S 架构
我们的项目属于 **B/S 架构**（Browser/Server，浏览器/服务器模式）。
*   **通俗理解**：用户只需要一个浏览器（Chrome/Edge）就能访问，不需要像打游戏一样下载安装包。
*   **面试题**：*请简述 B/S 架构与 C/S 架构的区别？*
    *   **答**：B/S 是浏览器对服务器，维护方便，分布性强；C/S 是客户端（App/exe）对服务器，响应速度快但需要安装更新。本项目采用 B/S 架构，便于学生随时随地访问。

### 1.2 前后端分离
*   **现状**：你的项目代码分成了两堆：
    1.  **前端（Frontend）**：`web1/xingyingmei` 根目录下的大部分 `.vue`, `.js` 文件。负责“长得好看”和“用户交互”。
    2.  **后端（Backend）**：`src/main/java` 下的 Java 代码。负责“数据处理”和“业务逻辑”。
*   **原理**：前端和后端通过 **HTTP 接口（API）** 说话。前端像是餐厅服务员（递菜单、端菜），后端像是厨房大师傅（做菜）。
*   **面试题**：*什么是前后端分离？为什么要这么做？*
    *   **答**：前后端分离是指前端负责页面展示，后端负责数据接口。好处是：职责清晰，开发效率高（互不干扰），且同一个后端可以同时服务网页端和手机 App。

## 第二章：前端世界 —— Vue.js 与页面渲染

### 2.1 核心技术：Vue 3 + Vite
你的项目使用了 **Vue.js** 框架。
*   **文件特征**：`.vue` 文件。
*   **结构**：每个 `.vue` 文件都是一个组件，包含三部分：
    *   `<template>`：HTML 骨架（页面的结构）。
    *   `<script>`：JavaScript 逻辑（页面的行为，如点击按钮发生什么）。
    *   `<style>`：CSS 样式（页面的化妆）。

### 2.2 关键原理：SPA（单页应用）
*   **现象**：你在网站里点来点去，页面不会像老网站那样“白屏闪烁”一下，而是瞬间切换。
*   **原理**：整个网站其实只有一个 HTML 页面（`index.html`）。当你点击菜单时，**Vue Router**（路由管理器）只是把旧的组件（页面内容）拆下来，换上新的组件。
*   **代码对应**：`src/router/index.js` 定义了哪个网址对应哪个组件。
*   **面试题**：*什么是 SPA？优缺点是什么？*
    *   **答**：SPA 是单页应用。**优点**：用户体验好，流畅，减轻服务器压力。**缺点**：首屏加载慢（第一次要把所有逻辑加载进来），SEO（搜索引擎优化）相对较难。

### 2.3 状态管理（数据怎么传递）
*   **场景**：登录后，所有页面都要知道“我是谁”。
*   **原理**：使用 **Session/LocalStorage** 或状态管理库。你的项目在 `src/utils/sessionManager.js` 里把用户信息存到了浏览器的 LocalStorage 里。

## 第三章：后端大脑 —— Spring Boot 与 MVC

### 3.1 核心技术：Spring Boot
*   **地位**：Java 界的“瑞士军刀”，简化了企业级开发。
*   **面试题**：*什么是 Spring Boot？*
    *   **答**：它是基于 Spring 框架的脚手架，核心特点是“约定大于配置”和“自动装配”，能让你快速启动一个 Web 服务，不用写繁琐的 XML 配置文件。

### 3.2 代码结构：MVC 模式（变种）
你的后端代码严格分层，这是面试必考的**分层架构**：

1.  **Controller 层（控制层/接口层）**
    *   **位置**：`com.study.controller` 或 `com.study.service.*.*Controller`
    *   **职责**：**前台接待**。接收前端发来的请求，检查参数，然后指挥 Service 干活。
    *   **代码示例**：`MatchingController.java` 里的 `@GetMapping`。
2.  **Service 层（业务逻辑层）**
    *   **位置**：`com.study.service`
    *   **职责**：**核心员工**。处理复杂的业务（比如：根据 GPA 算分、判断是否符合入学要求）。
    *   **代码示例**：`MatchingResultService.java`。
3.  **DAO / Mapper 层（数据访问层）**
    *   **位置**：`com.study.mapper`
    *   **职责**：**仓管员**。只负责跟数据库打交道（增删改查 SQL）。
    *   **代码示例**：`UserMapper.java`。

*   **面试题**：*请描述 Spring MVC 的请求处理流程？*
    *   **通俗答**：请求先到 DispatcherServlet（总机），转发给 Controller（分机），Controller 调用 Service（办事员），Service 让 Mapper（仓管）去 Database（仓库）拿数据，最后原路返回给用户。

## 第四章：数据流转 —— 它们怎么聊天？

### 4.1 通信协议：HTTP
*   **请求（Request）**：前端发给后端。包含 URL、Header（身份信息）、Body（数据）。
*   **响应（Response）**：后端回给前端。包含状态码（200 OK, 404 Not Found, 500 Error）和数据（JSON）。

### 4.2 数据格式：JSON
*   **样子**：`{ "username": "xiaoming", "age": 22 }`
*   **作用**：前后端的“普通话”。Java 对象在传输前变成 JSON 字符串，Vue 收到后把 JSON 变回 JS 对象。

## 第五章：本项目核心业务深度解析（面试亮点）

### 5.1 亮点一：AI 智能选校算法
*   **原理**：这不是简单的数据库查询，而是**策略模式（Strategy Pattern）**的应用。
*   **代码逻辑**：
    1.  `MatchingAlgorithmManager` 根据地区（如“英国”）动态选择一组算法策略。
    2.  每个学校（如伯明翰、埃克塞特）都有一个独立的算法类（继承自 `SchoolMatchingStrategy`）。
    3.  算法内部根据 GPA、专业背景计算 `matchScore`（匹配分）。
*   **面试话术**：*“我的项目中使用了策略模式来实现选校算法。针对不同学校的录取规则，封装了不同的策略类，这样如果要加新学校，只需要加一个类，不需要修改原有代码，符合**开闭原则**。”*

### 5.2 亮点二：登录与鉴权
*   **流程**：
    1.  前端输入密码 -> 2. 后端 `LoginController` 接收 -> 3. `BCryptPasswordEncoder` 校验密码（**注意：数据库里存的是加密后的乱码，不是明文，这叫加盐哈希**） -> 4. 校验通过，写入 `HttpSession`。
*   **拦截器**：后端配置了拦截器（Interceptor），除了登录接口外，其他接口都会检查 Session 里有没有人，没人就驳回（401 Unauthorized）。

## 第六章：数据库与运维（地基）

### 6.1 MySQL 数据库
*   **表结构**：
    *   `users`：存账号。
    *   `students`：存详细档案。
    *   `matching_results`：存选校结果。
*   **面试题**：*数据库三大范式是什么？索引是做什么的？*
    *   **通俗答**：索引就像书的目录，为了查数据更快。

### 6.2 部署（Deployment）
*   **流程**：
    *   前端：运行 `npm run build` 生成 `dist` 文件夹（一堆静态 HTML/CSS/JS）。
    *   后端：运行 `mvn package` 生成 `.jar` 包。
    *   服务器：用 Nginx 代理前端，用 Java 运行后端，两者配合工作。
