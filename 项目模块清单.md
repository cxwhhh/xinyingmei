#### 2.2.2 路由与会话

- `src/router/index.js`  
  链接：[index.js](file:///Users/chenxiwen/code/web1/xingyingmei/src/router/index.js)  
  - 使用内容：Vue Router、`sessionManager`
  - 对外导出：`export default router`
  - 关键方法/行为：
    - `routes[]` 定义了官网、用户页、管理端、AI选校等路由
    - `router.beforeEach`：
      - 使用 `sessionManager.isLoggedIn()`、`sessionManager.getUserInfo()`
      - 若 `meta.requiresAuth` 且未登录：`sessionManager.clearSession()` 后重定向 `/login`
      - 已登录但需要鉴权：调用 `await sessionManager.checkSession()`
    - `router.afterEach`：移除 `page-transitioning` class

- `src/utils/sessionManager.js`  
  链接：[sessionManager.js](file:///Users/chenxiwen/code/web1/xingyingmei/src/utils/sessionManager.js)  
  - 使用内容：`router`、`request`（见 `utils/request.js`）
  - 核心类：`class SessionManager`
  - 对外导出：`export default sessionManager`（单例）
  - 关键方法：
    - `startSessionCheck()`：每 5 分钟轮询 `checkSession()`，并在页面重新可见时触发检查
    - `stopSessionCheck()`：停止轮询
    - `checkSession()`：当本地 `isLoggedIn/userInfo` 存在时，请求 `GET /user/current` 验证会话；401 或 “用户未登录” 则清理并重定向
    - `setSession(userInfo)`：写入 `localStorage` 并启动检查，触发 `session-changed` 事件
    - `clearSession()`：清理本地登录态并停止检查，触发 `session-changed`
    - `updateUserInfo(partial)`、`getUserInfo()`、`isLoggedIn()`、`getUserId()`
  - 主要调用接口：`GET /api/user/current`（通过 `request.get('/user/current')`）

#### 2.2.3 统一请求客户端

- `src/utils/request.js`  
  链接：[request.js](file:///Users/chenxiwen/code/web1/xingyingmei/src/utils/request.js)  
  - 使用内容：Axios、Element Plus `ElMessage`（当前文件引入但未实际调用）、`router`
  - 对外导出：`export default request`（Axios 实例）
  - 关键配置：
    - `baseURL: '/api'`、`withCredentials: true`、`timeout: 60000`
  - 请求拦截器：
    - 若调用方传入了 `/api/...`，会剥掉前缀避免双重 `/api`
    - 清理 `params` 中的 `null/undefined`
  - 响应拦截器（关键约定）：
    - 取 `const res = response.data` 返回给业务层（不是返回 Axios Response）
    - 当 `res.code === 200` 或 `res.success === true`：认为成功
    - 当 “用户未登录”：清理 `localStorage` 并跳转 `/login`
    - 网络错误：统一修正超时/服务端 message，并在 401 时跳转 `/login`
